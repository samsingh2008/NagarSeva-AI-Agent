KEY FOLDERS EXPLAINED
====================

CLIENT SIDE (Next.js Frontend)
==============================

📁 app/
  └─ Next.js App Router directory. Contains all page routes and layouts.
     Use (routes) folder to group related pages together.

📁 components/
  ├─ common/ → Shared layout components used across multiple pages
  └─ ui/     → Reusable atomic UI components (Button, Modal, etc.)

📁 hooks/
  └─ Custom React hooks for logic reuse and state management
     Examples: useApi, useAuth, useFetch, useLocalStorage

📁 lib/
  └─ Utility libraries and API client setup
     Example: axios instance, API wrapper functions

📁 styles/
  └─ Global CSS and Tailwind directives
     All component-specific styles go in Tailwind classes

📁 utils/
  └─ Pure utility functions (formatters, validators, helpers)


SERVER SIDE (Express Backend)
=============================

📁 routes/
  └─ Define API endpoints and route handling
     Organize by feature (userRoutes, issueRoutes, etc.)

📁 controllers/
  └─ Handle HTTP requests and responses
     Call appropriate services for business logic

📁 models/
  └─ MongoDB schemas using Mongoose
     Define data structure and relationships

📁 services/
  ├─ Contain business logic (rules, calculations, etc.)
  ├─ Interact with models and databases
  ├─ geminiService.ts ⭐ → Gemini AI integration
  └─ Reusable across multiple controllers

📁 middleware/
  └─ Express middleware for cross-cutting concerns
     Auth, validation, error handling, logging

📁 config/
  ├─ database.ts ⭐ → MongoDB connection setup
  ├─ gemini.ts ⭐   → Gemini AI initialization
  └─ Other configuration files

📁 utils/
  └─ Helper functions used across services
     Validators, formatters, error types, loggers

📁 schemas/
  └─ Request validation schemas (Joi, Zod, etc.)
     Validate input before processing


SPECIAL FILES
=============

⭐ src/index.ts              → Express app setup & server startup
⭐ config/database.ts        → MongoDB connection (Mongoose setup)
⭐ config/gemini.ts          → Google Gemini AI initialization
⭐ services/geminiService.ts → AI-related business logic


CONFIGURATION FILES
===================

Client:
  - tsconfig.json      → TypeScript configuration with path aliases
  - tailwind.config.js → Tailwind CSS theme & plugins
  - postcss.config.js  → PostCSS configuration (for Tailwind)
  - next.config.js     → Next.js optimization & settings
  - package.json       → Dependencies & scripts

Server:
  - tsconfig.json      → TypeScript configuration
  - .eslintrc.json     → Code linting rules
  - package.json       → Dependencies & scripts
  - .env (root)        → Environment variables

Root:
  - .gitignore         → Files to exclude from git
  - .env.example       → Template for environment variables
  - package.json       → Workspace configuration


DEVELOPMENT WORKFLOW
====================

1. Create feature branch:
   git checkout -b feature/my-feature

2. Make changes:
   - Frontend: Edit components in client/
   - Backend: Edit services/controllers in server/

3. Test locally:
   npm run dev (from root)

4. Commit changes:
   git add .
   git commit -m "feat: description"

5. Push & create PR
